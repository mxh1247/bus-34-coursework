Persona Name:
- Healthcare professional
Role/User Type:
- Employee of healthcare provider
Goals:
- Quickly review user inputs (health info)
- Make informed medical decisions based on user inputs
- Informing those users of their decisions/conclusions
- Efficiently manage user profiles, adding or removing users and their info when necessary
- Ensure confidential information remains secure
Pain Points:
- Busy schedule
- Limited time per patient
- Outdated patient information
- Need to adhere to legal restrictions, such as removing patient info when no longer needed
Related User Stories (IDs):
- S1
- S2
- S4

S1
Features:
- View user inputs
Story Title:
- View user health information
User Story:
- Needs to make decisions based on user input, but may be short on time, so information should be displayed completely but concisely
Acceptance Criteria:
- List of patients
- When clicking on a patient their data is displayed
- Time info was changed is displayed
Priority (Must/Should/Could):
- Must
Justification of prioritisation/Trade-off:
- Fundamental part of the app, healthcare professionals need to be able to view information to make decisions on it
Story Owner:
- Matt

S4
Features:
- Manage patient
Story Title:
- User changes healthcare provider, info is no longer needed, or info needs updating manually on the healthcare professional's end
User Story:
- Wants to maintain up to date and accurate patient records
Acceptance Criteria:
- Button to update user when viewing them from the list described in S1
- Update user button opens panel displaying each info field as updatable text boxes, additional button for deleting a user.
- Button to save changes, when clicking prompted with a -reason for change- textbox, which, when submitted, updates the user database entry with the information and records the time of update and reason for update.
Priority (Must/Should/Could):
- Must
Justification of prioritisation/Trade-off:
- Needed to ensure all information and patient lists can be kept up to date and accurate
Story Owner:
- Matt

S2
Features:
- Data Visualisation
Story Title:
- View health trend graph to more easily visualise changes in health
User Story:
- As a healthcare professional monitoring multiple patients, I want to view clear, clinically useful graphs of a patient’s logged health data, so that I can quickly identify trends, detect early warning signs, and make informed decisions about their care.
Acceptance Criteria:
- Given the system has stored patient health data with time stamps when the healthcare professional opens the graph view then the graph should display accurate, time‑stamped data directly from the patient’s logs
- Given the healthcare professional accesses the graph when the system retrieves and processes the patient’s data then the graph should load quickly without requiring any manual data manipulation from the use
Priority (Must/Should/Could):
- Should
Justification of prioritisation/Trade-off:
- This feature directly supports clinical workflow, enhances decision‑making, and improves the quality and safety of patient care. It transforms raw data into actionable insight, which is exactly what healthcare professionals need.
Story Owner:
- Matt
